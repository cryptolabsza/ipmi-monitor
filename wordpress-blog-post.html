Why We Built This

Managing a datacenter or colocation facility means keeping track of dozens‚Äîsometimes hundreds‚Äîof servers. When hardware issues occur, you need to know immediately. Traditional IPMI tools require logging into each server's BMC individually, which doesn't scale.

IPMI Monitor solves this by providing a centralized dashboard that:

  • Collects IPMI System Event Logs (SEL) from all servers automatically
  • Shows real-time sensor data (temperature, fans, voltage, power)
  • Identifies exactly which DIMM has ECC memory errors
  • Classifies events by severity (Critical, Warning, Info)
  • Integrates with Prometheus and Grafana for alerting

Key Features

üîç Automated Event Collection

IPMI Monitor uses 10 parallel workers to collect SEL logs from your entire fleet in minutes, not hours. Set it and forget it‚Äîevents are collected automatically every 5 minutes (configurable).

üìä Real-Time Dashboard

The auto-refreshing dashboard shows server status at a glance. See which servers have critical events, warnings, or are running smoothly. Filter by time range from 1 hour to 30 days.

üå°Ô∏è Live Sensor Monitoring

View real-time temperature, fan speed, voltage, and power readings for each server. Spot thermal issues before they cause downtime.

üíæ ECC Memory Error Tracking

When ECC errors occur, IPMI Monitor tells you exactly which memory sensor detected them (e.g., CPU1_ECC1). Know which DIMM to replace before it fails completely.

üîê Admin Authentication

Two access levels: read-only for operators (view dashboard, export CSV) and admin access for sensitive operations (add/delete servers, clear SEL logs, manage credentials).

üìà Prometheus & Grafana Integration

Native /metrics endpoint for Prometheus scraping. We include a pre-built Grafana dashboard JSON file so you can set up alerting in minutes.

Quick Start

Deploy IPMI Monitor in under 5 minutes with Docker:

# Quick Start with Docker
docker run -d \
  --name ipmi-monitor \
  -p 5001:5000 \
  -e IPMI_USER=admin \
  -e IPMI_PASS=YourIPMIPassword \
  -e ADMIN_USER=admin \
  -e ADMIN_PASS=YourSecurePassword \
  -v ipmi_data:/app/instance \
  ghcr.io/jjziets/ipmi-monitor:latest

# Access at http://localhost:5001

Or use Docker Compose for production deployments:

# docker-compose.yml
version: '3.8'
services:
  ipmi-monitor:
    image: ghcr.io/jjziets/ipmi-monitor:latest
    container_name: ipmi-monitor
    restart: unless-stopped
    ports:
      - "5001:5000"
    environment:
      - IPMI_USER=admin
      - IPMI_PASS=${IPMI_PASS}
      - ADMIN_USER=admin
      - ADMIN_PASS=${ADMIN_PASS}
      - SECRET_KEY=${SECRET_KEY}
    volumes:
      - ipmi_data:/app/instance

volumes:
  ipmi_data:

Prometheus Metrics

IPMI Monitor exposes these metrics at /metrics:

  • ipmi_server_reachable - BMC reachability (1=yes, 0=no)
  • ipmi_server_power_on - Server power status
  • ipmi_events_total - Total events per server
  • ipmi_events_critical_24h - Critical events in last 24h
  • ipmi_events_warning_24h - Warning events in last 24h
  • ipmi_total_servers - Total monitored servers
  • ipmi_reachable_servers - Currently reachable servers

Add to your Prometheus config:

scrape_configs:
  - job_name: 'ipmi-monitor'
    static_configs:
      - targets: ['your-server:5001']
    scrape_interval: 60s
