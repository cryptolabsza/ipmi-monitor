üîÑ GPU Soft Reset SSH Required

üîç When this triggers:
  • GPU stopped responding to commands
  • Driver reports GPU needs recovery
  • GPU exceptions during compute operations
‚öôÔ∏è What it does:
  1. Workload must be stopped first (auto or manual)
  2. Runs nvidia-smi --gpu-reset -i <gpu>

‚ö†Ô∏è Note: Requires no active processes on the GPU. Stop Workload runs first.
üìâ GPU Clock Limit SSH Required
Reduce GPU clocks by 20% to improve stability
üîç When this triggers:
  • Recurring GPU errors (same error multiple times)
  • High rate of correctable memory errors (ECC)
  • GPU showing signs of instability or overheating
‚öôÔ∏è What it does:
  1. Gets current max GPU clock speed
  2. Sets new limit at 80% of max (20% reduction)
  3. Makes it persistent across reboots (via crontab)
  4. Flags GPU for maintenance review
‚ö†Ô∏è Trade-off: Reduces performance by ~15-20% but improves stability. Good for aging GPUs.
These actions may cause brief interruption to other workloads on the same system.
üîå PCI Bus Reset SSH Required

üîç When this triggers:
  • GPU "fell off the bus" - disconnected from system

  • GPU not responding to any driver commands
‚öôÔ∏è What it does:

  2. Removes GPU from PCI bus (echo 1 > .../remove)
  3. Rescans PCI bus to re-detect GPU (echo 1 > .../rescan)
  4. Waits ~30 seconds for GPU to reinitialize
  5. If successful, container/VM can be restarted
